sparseCaseOptimisation :: TypeEnv -> Exp -> Exp Source #

Sparse case optimisation uses ana to transform the program from top to down. It checks if the given alterntive can be removed. The alternative can be removed if it does not match any of the Nodes in the type associated with the variable. Literal matching alternatives must be kept.
